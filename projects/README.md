# Projects Root

This folder is reserved for future projects so Git and workflow boundaries stay clean.

## Current Rule

- the current SOOP/DC workflow project remains at `C:\python`
- it is treated as the first standalone project in the portfolio
- future projects should be created under `C:\python\projects\` or in their own repositories

## Why The Current Project Stays In Place

The existing workflow files and prompts already assume `C:\python` paths. Leaving the current project in place avoids breaking the harness while still establishing a clean multi-project convention for later work.

## Recommended Naming

- `C:\python\projects\project-a`
- `C:\python\projects\project-b`

or separate repositories with matching folder names.
